# Dynamic CMS Summary

This work happened in the `worktree-dynamic-content-cms` branch and turned the non-library public site pages into CMS-backed pages while leaving the library branch of the site alone. The CMS was rebased onto `development`, configured for the dev URL `https://ssedev.jjalangtry.com`, seeded with the current static pages, and deployed through the local `ssequel-cms-dev.service` on port `3130` behind Caddy/Cloudflare. OAuth for the dev domain needs the Google origin `https://ssedev.jjalangtry.com` and callback `https://ssedev.jjalangtry.com/api/auth/callback/google`; the seeded database includes `jjl4287@g.rit.edu` as a President/SE Office account for full CMS testing.

The CMS now supports editable page blocks that preserve the existing public-page layouts: sections, headings, markdown/text, images, image/text card rows, general card grids, app-backed widgets, CTAs, dividers, raw HTML, photo grids, and photo carousels. Pages can be created, edited, published/unpublished, restored, and controlled in the navbar, while locked system pages protect routes that should not be casually deleted. Existing static pages were translated into block content, and card/photo features were expanded so pages can rotate photos from historian categories and optionally reveal sections or card rows with scroll-fade animations.

The Pages dashboard UI was tightened after screenshot review: the page list is now a bounded scroll panel, the page editor has an independently scrollable block list and editor body, mobile action controls wrap instead of clipping, and editor forms use responsive grids so fields do not overflow small screens. Verification used Playwright screenshots and a short screen recording served through Tailscale artifacts, plus `typecheck`, `lint`, focused CMS tests, and a production build. The live dev service was restarted successfully, and Uptime Kuma already has an active monitor for `ssedev.jjalangtry.com`.
